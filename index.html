{DFR} Declarative Form Rendering
A Shadow Smith Project

Build Complex Forms Without the Boilerplate.

A headless web component library for building dynamic forms through pure schema configuration. Eliminate repetitive UI code and gain full styling control.

The Old Way (Painful) <label for="email">Email *</label>
<input type="email" id="email" required onchange="validate()">
<span class="error-msg" style="display:none">Invalid email</span>
<script>
  // 50 lines of validation logic...
</script>
The DFR Way (Declarative) // JSON Schema
email: {
  label: "Email",
  required: true,
  pattern: "^[^\\s@]+..."
}

// HTML
<dfr-text-question
  question="formSchema.email">
</dfr-text-question>

How It Works

1

Define Your Form Schema

Create a form schema object defining the configuration for each question.

{
  effectiveDate: {
    label: "Effective Date",
    required: true,
    minDate: "2020-01-01",
    maxDate: "2025-12-31"
  }
}
2

Add Components to HTML

Use a question component and bind it to your question schema.

<dfr-date-question
  id="effectiveDate"
  question="effectiveDate">
</dfr-date-question>
3

Style with CSS Parts

Customize question styles using standard CSS selectors.

dfr-date-question::part(input) {
  border: 2px solid #00f2ff; /* Neon blue */
  border-radius: 6px;
  padding: 0.75rem;
}

dfr-date-question::part(label) {
  color: #ffffff;
  font-weight: 600;
}
