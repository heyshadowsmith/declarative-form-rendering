{DFR} Declarative Form Rendering

Make Complex Forms Simple.

Declarative Form Rendering (DFR) is a set of headless web components and question JSON schemas for building and managing complex, dynamic forms in a declarative way.

// Schema
driverAge: {
  label: "Driver's Age",
  placeholder: "25",
  description: "Must be 18+",
  required: true,
  disabled: false,
  min: 18,
  max: 100
}

// Rendering
<dfr-number-question question="driverAge">
</dfr-number-question>

// Reactivity
if (driverAge.value < 25) {
  driverAge.description = "Young driver fee applies"
}

How DFR Works

1

Define Your Form Schema

Create a form schema object defining your questions using Declarative Question Schemas.

{
  formSchema: {
    driverAge: {
      label: "Driver's Age",
      placeholder: "25",
      description: "Must be 18+",
      required: true,
      disabled: false,
      min: 18,
      max: 100
    },
    rentalDate: {
      label: "Pick-up Date",
      description: "Select a date",
      required: true,
      disabled: false,
      minDate: "2024-01-01",
      maxDate: "2025-12-31"
    }
  }
}
2

Add Web Components to HTML

Use the question components that match each of your question types and bind them to your question schemas.

<dfr-number-question
  id="driverAge"
  question="formSchema.driverAge">
</dfr-number-question>

<dfr-date-question
  id="rentalDate"
  question="formSchema.rentalDate">
</dfr-date-question>
3

Style with CSS Parts

Customize question styles using standard CSS selectors.

dfr-date-question::part(input) {
  border: 2px solid #00f2ff;
  border-radius: 6px;
  padding: 0.75rem;
}

dfr-date-question::part(label) {
  color: #ffffff;
  font-weight: 600;
}
4

Update Your Form Schema

Simply update your schema object based on business logic. The form automatically rerenders to reflect changes.

// Add fee warning for young drivers
if (driverAge.value < 25) {
  formSchema.rentalDate.description = "Young driver fee applies";
} else {
  formSchema.rentalDate.description = null;
}
